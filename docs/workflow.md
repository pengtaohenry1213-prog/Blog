# 开发与协作流程

## 一、分支管理
- `main`：生产环境代码（受保护，仅通过PR合并）
- `develop`：开发环境主分支（功能分支完成后合并至此）
- 功能分支：`feature/xxx`（从develop分出，命名格式：feature/功能名称）

## 二、环境配置
- 开发环境：本地启动前端、后端、MySQL、Redis（需单独配置）
- 测试环境：通过docker-compose启动全部容器
- 生产环境：云服务器部署，配置监控与日志收集

## 三、CI/CD流程（可选）
1. 代码提交触发自动构建
2. 自动化测试通过后推送镜像至仓库
3. 远程服务器拉取镜像并重启容器
